## **AI 멀티 에이전트 시스템 설계 문서: F&B Startup Navigator**

### 1. 시스템 개요 (System Overview)

**F&B Startup Navigator**는 예비 요식업 창업자의 개인 데이터와 실제 시장 데이터를 종합적으로 분석하여, 성공 가능성이 높은 맞춤형 창업 아이템과 실행 로드맵을 3가지 제안하는 지능형 의사결정 지원 시스템입니다.

- **목표:** 데이터 기반의 객관적인 분석을 통해 창업 실패율을 낮추고, 예비 창업자가 자신에게 최적화된 비즈니스 모델을 발견하도록 돕습니다.
- **핵심 특징:** 각기 다른 전문성을 가진 AI 에이전트들이 협업하여, 인간 전문가의 컨설팅 과정을 모방하고 자동화합니다. LLM이 부족한 실시간 시장 분석, 상권 데이터 분석 등은 외부 도구(Tool)를 적극적으로 활용하여 신뢰성을 확보합니다.

### 2. 시스템 아키텍처 (System Architecture)

시스템은 총 5개의 전문 에이전트와 이들을 총괄하는 1개의 마스터 에이전트로 구성됩니다.



**워크플로우:**
1.  **사용자 입력**: '사전 데이터' JSON 형식으로 입력
2.  **Master Orchestrator**: 워크플로우 시작, 각 에이전트에게 순차적으로 작업 지시 및 결과 취합
3.  **Profiler Agent**: 사용자 데이터 분석하여 '창업자 페르소나' 생성
4.  **Market Analyst Agent**: '창업자 페르소나'와 '프로젝트 정보'를 바탕으로 외부 데이터 분석
5.  **Idea Validator Agent**: 분석 결과를 종합하여 가장 유망한 창업 아이템 3가지 선정
6.  **Roadmap Architect Agent**: 선정된 3가지 아이템 각각에 대한 상세 로드맵 생성
7.  **Report Synthesizer Agent**: 모든 결과를 최종 보고서 형태로 종합하여 사용자에게 제공

### 3. 에이전트 상세 설계 (Detailed Agent Design)

#### 3.1. Master Orchestrator (총괄 관리 에이전트)
- **역할**: 전체 프로세스를 조율하고 에이전트 간의 데이터 흐름을 관리하는 컨트롤 타워.
- **입력**: 사용자의 '사전 데이터' 원본.
- **처리**:
    - 각 에이전트를 순서대로 호출.
    - 이전 에이전트의 출력물을 다음 에이전트의 입력물로 가공하여 전달.
    - 최종적으로 모든 결과물을 취합하여 Report Synthesizer에게 전달.
- **사용 도구**: 없음 (내부 로직 및 상태 관리).
- **출력**: 최종 보고서.

#### 3.2. Profiler Agent (창업자 프로파일링 에이전트)
- **역할**: 사용자의 `personalInfo`를 분석하여 창업 성향, 강점, 약점, 리스크 수용도 등을 포함하는 '창업자 페르소나'를 정의.
- **입력**: `personalInfo` JSON 객체.
- **처리**:
    - **성별, 나이**: 타겟 고객층 설정에 대한 잠재적 선호도 추론.
    - **MBTI**: 업무 스타일, 고객 응대 방식, 마케팅 선호 방향성 추론 (예: J는 체계적 운영, P는 유연한 메뉴 개발에 강점).
    - **previousCompany**: 이전 경력을 통해 얻은 역량 분석 (예: 대기업 출신 -> 시스템/프로세스 구축 능력, 마케팅 직무 -> 브랜딩/홍보 능력).
    - **startupExperience**: `true`일 경우, 리스크 수용도가 높고 변화에 유연하다고 판단. `false`일 경우, 안정적인 프랜차이즈나 운영 난이도가 낮은 아이템을 추천 가중치에 반영.
- **사용 도구**: 없음 (LLM의 추론 능력 활용).
- **출력**: '창업자 페르소나' 요약 보고서 (텍스트).
    ```json
    {
      "personaSummary": "30대 여성, 스타트업 경험 보유. 체계적(J)이면서도 창의적인(N) 성향. 마케팅 직무 경험을 살려 브랜딩에 강점을 보일 것으로 예상. 리스크 수용도 '중상'.",
      "recommendedStyle": ["브랜딩 중요", "체계적 운영", "고객 경험 중시"],
      "riskTolerance": "High"
    }
    ```

#### 3.3. Market Analyst Agent (시장 분석 에이전트)
- **역할**: LLM이 가장 취약한 영역으로, **외부 데이터 도구를 적극 활용**하여 지정된 지역의 상권을 객관적으로 분석.
- **입력**: `projectInfo` (foodSector, region, capital), '창업자 페르소나'.
- **처리**:
    1.  사용자가 지정한 `region` (예: "마포구")에 대해 **상권 분석 API** 호출.
        - 유동인구 (연령/성별/시간대별), 주변 직장인/거주인구 특성, 평균 임대료 시세 분석.
    2.  사용자가 지정한 `foodSector` 또는 전체 `foodSector`에 대해 **경쟁자 분석 API** 호출.
        - 해당 지역 내 동종/유사 업종의 밀집도, 프랜차이즈 비율, 평균 평점(리뷰) 분석.
    3.  **소셜/트렌드 분석 도구**를 활용하여 최근 서울 및 해당 지역의 외식 트렌드 키워드 추출.
- **사용 도구**:
    - **Tool 1: 상권 정보 시스템 API (정부 제공 데이터 등)**: 유동인구, 배후지 인구 통계 제공.
    - **Tool 2: 네이버/카카오 지도 API**: 특정 지역 내 음식점 카테고리별 업체 수, 위치, 평점 등 크롤링/분석.
    - **Tool 3: 소셜 미디어 트렌드 분석기 (Instagram, Blog 등)**: '#마포맛집' 등 키워드 분석으로 최신 트렌드 파악.
- **출력**: 시장 분석 데이터 보고서 (정량적 데이터).
    ```json
    {
      "region": "마포구",
      "demographics": "20-30대 직장인 및 대학생 유동인구 많음",
      "avgRent": "평당 25만원",
      "competitorDensity": {"일식": "높음", "중식": "보통", "양식": "높음"},
      "emergingTrends": ["수제버거", "위스키바", "베이글"]
    }
    ```

#### 3.4. Idea Validator Agent (창업 아이템 검증 에이전트)
- **역할**: '창업자 페르소나'와 '시장 분석 데이터'를 결합하여, 제공된 `foodSector` 목록 내에서 가장 성공 확률이 높은 구체적인 창업 아이템 3가지를 선정하고 순위화.
- **입력**: '창업자 페르소나', '시장 분석 데이터', 사용자의 `projectInfo`.
- **처리**:
    - `foodSector` 내 모든 세부 아이템을 후보군으로 설정.
    - 각 후보 아이템에 대해 점수화 모델(Scoring Model) 적용.
        - **시장성(40%)**: 트렌드 부합성, 경쟁 강도 (낮을수록 좋음).
        - **사용자 적합성(30%)**: 창업자 페르소나와의 부합도, 운영 난이도.
        - **수익성(30%)**: 예상 자본금 대비 기대 수익, 객단가.
    - 최고점 순으로 3개 아이템 선정. 각 아이템에 대한 선정 이유를 명확히 서술.
- **사용 도구**: 없음 (LLM의 복합적 추론 및 평가 능력 활용).
- **출력**: 추천 아이템 TOP 3 리스트.
    ```json
    [
      {"rank": 1, "item": "베이글 전문점", "reason": "마포구 2030 직장인 타겟. 최근 트렌드 부합. 스타트업 경험자의 브랜딩 능력 활용 시 차별화 가능. 경쟁 강도 '중하'.", "concept": "뉴욕 스타일의 정통 베이글과 크림치즈"},
      {"rank": 2, "item": "위스키, 몰트바", "reason": "...", "concept": "..."},
      {"rank": 3, "item": "일본식 덮밥(돈부리) 전문점", "reason": "...", "concept": "..."}
    ]
    ```

#### 3.5. Roadmap Architect Agent (로드맵 설계 에이전트)
- **역할**: 선정된 3가지 아이템 각각에 대해, 사용자가 제공한 '로드맵 구성' 템플릿에 맞춰 구체적이고 실행 가능한 계획을 수립.
- **입력**: 추천 아이템 TOP 3 리스트, 사용자의 원본 데이터(`personalInfo`, `projectInfo`).
- **처리**:
    - 3개 아이템에 대해 각각 아래의 로드맵 생성.
    - **공간 기획**: 아이템 콘셉트(예: "뉴욕 베이글")에 맞는 인테리어, 간판 키워드 도출 후 **이미지 생성 AI** 호출하여 시안 생성.
    - **운영 준비**: '베이글'에 필요한 밀가루, 크림치즈 등 주요 식자재 공급업체 유형 추천. 콘셉트에 맞는 패키지 디자인 아이디어 및 **이미지 생성 AI** 시안 제공.
    - **자금 계획**: 사용자 `capital`을 바탕으로 부족한 자금 확보를 위한 **정책자금 검색 도구** 활용. (예: "청년창업사관학교", "소상공인 정책자금").
    - **입지 선정**: '시장 분석 데이터'를 바탕으로 "마포구 내 연남동, 합정역 인근"과 같이 구체적인 추천 입지 조건 제시.
    - **행정 및 인허가**: '일반음식점'에 필요한 사업자등록, 영업신고, 위생교육 등 필수 절차 목록화.
    - **메뉴 개발**: '시그니처 메뉴'(예: 쪽파 크림치즈 베이글, 잠봉뵈르 베이글)와 가격 전략 제안. **이미지 생성 AI**로 메뉴 사진 예시 생성.
- **사용 도구**:
    - **Tool 4: Text-to-Image AI (DALL-E 3, Midjourney API 등)**: 인테리어, 간판, 패키지, 메뉴 시안 생성.
    - **Tool 5: 정책자금 정보 API/스크래퍼**: 사용자의 조건에 맞는 정부 지원 사업 검색.
- **출력**: 3개의 상세 로드맵 객체.

#### 3.6. Report Synthesizer Agent (최종 보고서 생성 에이전트)
- **역할**: 각 에이전트로부터 생성된 결과물들을 취합하여, 사용자가 이해하기 쉬운 일관된 형식의 최종 보고서로 가공.
- **입력**: '창업자 페르소나', '시장 분석 데이터', '추천 아이템 TOP 3', '상세 로드맵 3개'.
- **처리**:
    - 서론: 시스템의 분석 과정 및 데이터 요약.
    - 본론 1: 창업자 페르소나 및 시장 분석 결과 브리핑.
    - 본론 2: 추천 아이템 3가지에 대한 비교 분석표 (장점, 단점, 예상 투자금, 추천 이유).
    - 본론 3: 각 아이템별 상세 로드맵 제시 (시각 자료 포함).
    - 결론: 최종 요약 및 창업자를 위한 조언.
- **사용 도구**: 없음 (LLM의 문서 작성 및 요약 능력 활용).
- **출력**: 사용자에게 전달될 최종 창업 컨설팅 보고서 (Markdown 또는 PDF 형식).

### 4. 신뢰성 및 데이터 기반 접근 전략

이 시스템의 핵심은 **LLM의 환각(Hallucination)을 최소화**하고 **신뢰도를 극대화**하는 것입니다.
- **역할 분리**: LLM은 '추론', '평가', '창의적 생성', '요약' 등 강점을 가진 영역에 집중합니다.
- **도구 우선주의**: '시장 데이터', '상권 분석', '트렌드' 등 사실 기반 정보가 필요한 영역은 LLM에게 직접 묻지 않고, 반드시 외부 API나 데이터베이스를 조회하는 도구를 통해 객관적인 데이터를 확보합니다.
- **근거 제시**: 모든 추천에는 "왜?"라는 질문에 답할 수 있도록 Profiler Agent와 Market Analyst Agent의 분석 결과를 명확한 근거로 제시합니다. 예를 들어, "마포구의 20대 여성 유동인구가 많고(데이터), 베이글에 대한 소셜 언급량이 급증하고 있어(트렌드) 베이글 전문점을 1순위로 추천합니다." 와 같이 서술합니다.
